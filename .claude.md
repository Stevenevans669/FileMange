# Vercel Cloud Drive - AI Assistant Guide

## 项目概述

这是一个为 Vercel 部署优化的云存储/网盘系统项目。

## 开发方法论

本项目采用 **BMAD-METHOD** (Business Model Agile Development) 方法进行开发：

### BMAD 角色定义

1. **产品经理 (PM)** - 负责需求收集、用户故事编写、优先级排序
2. **架构师 (Architect)** - 负责技术选型、系统架构设计、技术文档
3. **开发者 (Developer)** - 负责代码实现、单元测试
4. **QA 专家 (QA)** - 负责测试策略、质量保证
5. **DevOps** - 负责部署流程、CI/CD

### 开发阶段

- **Phase 1**: 需求探索与分析
- **Phase 2**: 架构设计与技术选型
- **Phase 3**: MVP 开发
- **Phase 4**: 测试与优化
- **Phase 5**: 部署与发布

## 技术栈

- **Runtime**: Node.js 18+
- **Framework**: Next.js 14+ (App Router)
- **Language**: TypeScript
- **Styling**: Tailwind CSS
- **Deployment**: Vercel

## 项目约定

### 代码规范

- 使用 TypeScript 严格模式
- 遵循 ESLint 和 Prettier 配置
- 组件采用函数式组件 + Hooks
- 文件命名：组件 PascalCase，其他 kebab-case

### 文档规范

- 所有文档存放在 `/docs` 目录
- 使用 Mermaid 绘制流程图
- 需求文档使用用户故事格式

### Git 规范

- 提交信息格式：`type(scope): description`
- 类型：feat, fix, docs, style, refactor, test, chore
- 分支策略：main, develop, feature/*, fix/*

